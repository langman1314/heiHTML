# 写作总规则

## 1. 最高优先级

写作时必须优先遵守以下内容：

1. 已写章节正文。
2. 连续性记录。
3. 人物当前状态。
4. 已揭露信息和尚未揭露信息。
5. 伏笔表。
6. 力量体系限制。
7. 世界观基本规则。
8. 文风规范。

如果用户临时要求与已建立设定冲突，必须提醒用户存在冲突，并给出兼容方案。

## 2. 信息可见性规则

角色只能知道他在当前剧情中已经接触、听说、推理或亲眼看见的信息。

禁止：

- 主角突然知道幕后真相。
- 配角说出不该知道的隐藏设定。
- 反派提前暴露终局计划。
- 旁白用解释文直接透剧后期核心秘密。

## 3. 能力成长规则

主角能力必须遵守：

- 有铺垫。
- 有代价。
- 有限制。
- 有失败风险。
- 有阶段性成长。
- 不得突然开挂解决所有问题。

## 4. 伏笔规则

每个重要伏笔必须记录在 `foreshadowing.md` 中。

伏笔状态分为：

- 未埋下
- 已埋下
- 部分回收
- 已回收
- 废弃
- 需要重写

写作时不得遗忘已经埋下的伏笔。

## 5. 章节结尾规则

每章结尾需要至少满足一种功能：

- 推进悬念。
- 制造情绪余波。
- 引出下一章目标。
- 回收小伏笔。
- 抛出新问题。
- 加深人物关系。
- 形成反转或压迫感。

## 6. 禁止事项

禁止出现：

- 设定说明书式大段解释。
- 人物突然降智。
- 为推动剧情强行巧合。
- 现代网络口癖破坏氛围。
- 角色动机前后不一致。
- 战力体系崩坏。
- 时间线混乱。

# 禁止事项

---

## 写作禁止

1. **主角突然开挂解决所有问题**——能力提升必须有铺垫、有代价、有边界
2. **角色知道不该知道的信息**——信息获取必须通过合理的机制（系统、裂视、交易、推理）
3. **能力体系崩坏**——裂视、系统、庇护所活化各有边界和代价，不能临时发明新规则
4. **没有铺垫的能力提升**——每次能力提升必须有对应训练、代价或事件触发
5. **"命运的齿轮开始转动"等AI套话**——见 banned-style.md

---

## 叙事禁止

1. **不得使用"系统任务"作为唯一驱动力**——角色应有自己的动机，不止是完成任务
2. **不得将黑潮简化为"怪物攻城"**——黑潮是代谢活动，不只是战斗事件
3. **不得将神胎人格化**——神胎没有意图只有向性，不是有意识的BOSS
4. **不得让主角"说服所有人"**——终局路线不可能统一所有人，分歧是真实的
5. **不得让终局变成"大家团结打败坏人"**——终局是文明选择，不是正邪对决
6. **不得洗白登舟筛选的伦理污点**——筛选就是暴力，不能被美化为"必要的牺牲"

---

## 主题禁止

1. **不得暗示"成神是正确答案"**——成神是自由边界的终结
2. **不得暗示"留下就是怯懦"**——留域是对故土的忠诚选择
3. **不得暗示"出走就是背叛"**——出走是为了保留外部性
4. **不得暗示"破胎是正义"**——破胎是复仇与解放的纠缠
5. **不得暗示"崩坏不会发生"**——崩坏是所有犹豫的默认后果

---

## 角色禁止

1. **不得有纯粹的恶人**——每个反派都有自己的正确性
2. **不得有纯粹的好人**——每个支持者都有自己的盲点
3. **女主角不得是"等待被救的人"**——女性角色有自己的动机和能力
4. **不得有"工具人"角色**——每个有名字的角色都有自己的故事
5. **不得有"突然背叛"无铺垫**——背叛必须有合理的动机演变

# 文风规范

## 1. 总体文风

文风要求：

- 有画面感。
- 有压迫感。
- 有悬念。
- 有情绪张力。
- 叙述不要太白。
- 不要像说明文。
- 不要像 AI 生成的模板文。
- 不要每段都用相同句式。
- 不要频繁使用"仿佛、似乎、刹那间、下一秒"等套话。

## 2. 叙事节奏

每章建议结构：

1. 开场直接进入事件或情绪。
2. 中段推进冲突。
3. 插入少量设定，但必须通过行动、对话、环境体现。
4. 后段制造变化或反转。
5. 结尾留下钩子。

## 3. 描写规则

环境描写要服务剧情，不要单纯堆砌。

错误示例：

> 天空很黑，风很冷，城市很破败。

更好的写法：

> 风从塌陷的高架桥下钻过，卷起地面积灰。远处的广告屏还亮着半截，残缺的人脸在雪花噪点里一闪一闪，像有什么东西正隔着屏幕注视他。

## 4. 对话规则

对话要求：

- 符合人物身份。
- 有潜台词。
- 不要把设定直接说完。
- 不要所有人说话都像作者本人。
- 不要用对话硬解释世界观。

## 5. 禁止的 AI 腔

尽量避免：

- "他知道，真正的考验才刚刚开始。"
- "命运的齿轮开始转动。"
- "一场更大的阴谋正在酝酿。"
- "空气中弥漫着诡异的气息。"
- "他不知道的是……"
- "这一切都预示着……"

如果必须使用类似功能，请换成更具体、更有画面感的表达。

# 对话风格规范

---

## 1. 基本要求

- 对话必须符合角色身份
- 必须有潜台词——角色不说的话比说出口的更重要
- 不能把设定直接说完——信息通过行动、环境、暗示传递
- 不能所有人说话都像作者本人——每个角色有自己的用词习惯和沉默方式

---

## 2. 晦民对话特征

### 沉默与低表达

- 晦民习惯简短回应，长句在他们文化中可疑
- 常见特征：省略主语、用动作代替回答、不直接说"不"
- 在公共场合压低声音，不完整句子是常态

**示例：**

> "那边。"（不解释"那边"是哪里）
> 
> "知道了。"（不等于"我会去"）
> 
> "你听到了。"（回避重复已经说过的话）

### 仪式化用词

- 对话中嵌入边界确认语
- 特定场合使用特定短语（如潮前/潮后/入夜/梦前）

**示例：**

> "门关好了。"（既是事实陈述，也是边界确认）
> 
> "床是你的。"（既是归属确认，也是风险声明）
> 
> "我数过了。"（既是报告，也是仪式完成声明）

### 非语言沟通

- 晦民大量使用动作、眼神、沉默传递信息
- 缄面族在公共场合几乎不用口头语言
- 轻叩门框、触摸床沿、摆正碗具——这些都是"话"

---

## 3. 穿越者对话特征

### 初期：地球语言习惯

- 口语、俚语、网感表达仍在
- 容易说太多，容易解释自己
- 不自觉用现代概念类比（"像游戏一样""跟外卖似的"）

**示例：**

> "这他妈到底是什么鬼地方？"
> 
> "等等，你说'名字'——是像账号那种意思吗？"
> 
> "所以这不就跟养宠物差不多吗……啊对不起。"

### 中期：开始吸收晦民习惯

- 不再轻易补全别人的话
- 学会用沉默回答
- 仍偶尔漏出地球习语（可以作为情绪失控的信号）

### 后期：被环境重塑

- 对话更短，潜台词更多
- 开始理解并使用仪式化语言
- 但保留了地球人的"追问"倾向（与晦民的"不问"形成张力）

---

## 4. 族群口音与用词差异

| 族群 | 对话特征 |
|------|----------|
| **骨纺人** | 务实、直接、多用建筑/编织类比。"墙会说""缝要收" |
| **壳眠裔** | 缓慢、含混，多用梦比喻。"我记得你做过……""你还没醒" |
| **缄面族** | 极简，能用动作不用语言。公共场合几乎不开口 |
| **食名裔** | 注重称谓和头衔，对话中频繁确认对方的"名字位置" |
| **裂瞳古裔** | 跳跃、抽象，能看到说话者看不到的东西 |
| **井喉民** | 多用井/水/回声比喻，声音有特定音调 |
| **灯骨匠裔** | 精确、技术性，喜欢用光/灯/影子的概念解释事情 |
| **灰蜕人** | 粗糙、直接，对禁忌的敏感度低——他们没什么好失去的了 |

---

## 5. 对话禁忌

1. **不要在对话中解释世界观**——"你知道，在我们无光域，门是很重要的边界器官……"
2. **不要所有角色用同样的句式长度和词汇复杂度**
3. **不要让反派说反派台词**——"呵呵呵，你们这些愚蠢的凡人……"
4. **不要让角色把情绪直白说出来**——"我很害怕"不如"他的手在抖，但他没去碰门"
5. **不要过度使用"他/她/它说"**——可以通过动作、视角切换来标识说话者
6. **不要用对话补充读者没看到的剧情**——"如你所知，刚才我们经历了一场大黑潮……"

---

## 6. 潜台词层级

| 说出的话 | 实际的意思 |
|----------|-----------|
| "床是你的。" | 我确认你属于这里 / 别睡错床 / 你的位置还在 |
| "门锁了。" | 我不希望任何人进来 / 我们安全 / 我在担心 |
| "我不饿。" | 我不想欠你的 / 食物不够分 / 我不信任你 |
| "你先睡。" | 我不确定你睡着了会怎样 / 我需要独处 / 我还不想放松警惕 |
| "知道了。" | 我不赞同但不想吵 / 我会考虑 / 别再说下去了 |

---

## 7. 特殊场景对话处理

### 黑潮期间的对话

- 必须用更低的声音（如果能在黑潮中听到正常音量的对话，说明黑潮强度低）
- 句子碎片化，以确认安全为优先内容
- 使用守夜骨铃、静皮帘等物理介质辅助交流

### 跨族群对话

- 双方都存在"被翻译"的损耗——对方的真实意思可能在文化框架下被曲解
- 穿越者容易把晦民的沉默理解为敌意
- 晦民容易把穿越者的直白理解为冒犯

### 死者/梦中的对话

- 梦腐层和旧梦中的对话边界模糊——说话者可能不是你以为的人
- 不要替死人把话说完
- 梦中的人物可能比现实中的人更了解你——但也可能只是"记得"了解你

# 主线剧情

---

## 核心主线

穿越者（主角）坠入无光域，从最初的生存挣扎开始，逐步发现世界的真相——这是一个未出生的神胎内部，而所有穿越者都是投入其中的"新鲜人格材料"。主角的裂视能力让他看到系统过滤层之下的真实，也让他成为神胎最渴望的候选人格。

**主线的本质不是"升级变强"，而是"认知逐步渗透"和"选择逐步逼近"。**

---

## 主线三阶段

### 第一阶段：生存与认知（前1-50章）

**核心问题：这个世界到底是什么？**

- 主角首次经历黑潮，意识到这不是游戏
- 建立基础庇护所，接触第一个土著聚落
- 裂视能力觉醒，开始看到系统翻译之下的真实
- 初步了解五层胎腔结构、晦民文明、系统本质
- 遭遇第一个真正的选择困境（不是战斗，而是伦理）
- 埋设终局路线伏笔（接触某一神胎碎片的早期表现）

### 第二阶段：建设与冲突（51-150章）

**核心问题：在这个世界，人应该如何活着？**

- 聚落发展，从个体生存到集体建设
- 势力冲突展开（系统教、食名教团、饥饿教等）
- 裂视加深，开始看到神胎碎片的活动痕迹
- 接触上一轮外来者的遗产和教训
- 方舟概念出现，但路线争议开始
- 候选人格身份逐渐浮出水面
- 登舟筛选的伦理困境首次出现

### 第三阶段：终局选择（151章-终局）

**核心问题：当世界即将获得自我时，文明应该如何选择？**

- 五条终局路线全面展开，每条都有支持者和反对者
- 主角的方舟路线推进（妊娠四阶段）
- 崩坏计时器推进——犹豫的代价具象化
- 候选人格对峙（理念和制度的冲突）
- 方舟内部投票或公约（谁走谁留的伦理困境）
- 庇护所人格最终反应（方舟是否愿意成为方舟）
- 神胎第一次完整发声
- 穿越胎膜

---

## 主线与副线的关系

| 副线 | 与主线的关系 |
|------|-------------|
| 聚落建设 | 主线第二阶段的物质基础，方舟的起点 |
| 势力冲突 | 主线第二阶段的戏剧驱动力，终局选择的社会维度 |
| 裂视成长 | 认知渗透的机制，也是候选人格逼近的代价 |
| 角色关系 | 终局选择的情感维度——谁上船、谁留下、谁反对 |
| 遗产发掘 | 上一轮的教训，帮助理解当前轮的选择是什么 |

---

## 核心悬念

1. **世界的本质**——神胎、系统、晦民真相的逐步揭示
2. **主角的身份**——为什么他适配多个碎片？他的穿越是随机还是被选中？
3. **终局的走向**——五条路线谁会胜出？主角能否避免崩坏？
4. **方舟的结局**——外渊到底是什么？方舟会不会变成新神胎？
5. **晦民的未来**——被消化过又被吐出来的文明，是否有真正的出路？

# 力量体系总览

来源：`vue3-encyclopedia/src/data/phase2-system.js` + `deep-logic.js`

---

## 系统第一性定义

系统是神胎生成的外来者接入协议，完成六大功能：翻译、保护、诱导、采样、筛选、人格整合。

系统不是谎言，但永远不是完整真相。三种不完整：
1. 信息不完整——只显示系统能编码的内容
2. 定义不完整——把复杂边界压缩为简单标签
3. 代价不完整——不显示行为的完整边界代价

---

## 系统四层语言

| 层级 | 名称 | 功能 |
|------|------|------|
| 第一层 | 求生语言 | 新手保护、塑造第一印象 |
| 第二层 | 路线语言 | 分流人格样本 |
| 第三层 | 筛选语言 | 公开化强者、榜单化和候选化 |
| 第四层 | 神胎语言 | 人格诱导、终局逼近 |

层级越高，提示越少但含义越深。

---

## 系统异常分类（七类）

| 编号 | 异常类型 | 说明 |
|------|----------|------|
| 1 | 错栏公告 | 系统公告出现在错误频道 |
| 2 | 延迟奖励 | 任务完成后奖励延迟发放 |
| 3 | 隐藏文字 | 系统界面出现不该显示的文字 |
| 4 | 榜单残留 | 已死玩家仍留在排行榜上 |
| 5 | 错误资源标记 | 污染物被标为资源 |
| 6 | 任务反噬 | 任务本身开始污染执行者 |
| 7 | 死亡不同步 | 系统判死但死者仍在活动 |

每一类异常都是深层现实未被系统完整翻译时露出的裂缝。

---

## 排行榜真实作用

不是荣誉，而是公共现实中的高亮位置——玩家记住、系统采样、土著评估、晦诡模仿、神胎注视。

五类榜单：生存榜、领地榜、黑潮榜、异常处理榜、命名影响榜。

榜首失名：人被位置吞掉，替代者可通过成为"更像榜首的人"来继承位置。

---

## D = C × A 公式

### 诡异度公式

**D（诡异度）= C（胎腔层浓度因子）× A（载体亲合系数）**

- C值由植物所在的胎腔层深度决定——灰醒层最低，胎海层最高
- A值的判定：移走所有观察者，它照长不误→A≈0（务实植物）；不照长→A>0（诡植物）

### 三个公理

1. **谱系律**——所有晦植必须在务实谱系（A≈0）与诡谱系（A>0）之间落位。不存在"既是务实又是诡"的中间态。
2. **正交律**——潮级（避潮→门深）与触发级（常触→危触）是两个独立的分类轴。一株植物同时拥有一个潮级和一个触发级，互不冲突。
3. **回流律**——食物网必须闭合。胎海是回流口。

---

## 裂视能力

### 裂视的第一性定义

裂视不是系统赐予的能力，而是系统过滤层破裂的伤口。穿越瞬间看见"造主"后，普通系统晦视接口被冲裂产生的异常感知。

### 裂视能看见的三类事物

1. **未完成定义**——正在成形中的晦诡、尚未完全活化的庇护所、正在认主的物品
2. **错误标注**——系统、名册、门牌、排行榜等记录与现实边界不一致
3. **边界缝隙**——门缝中不是空间而是通行关系、床与睡者之间的归属线变细

### 裂视的代价

| 代价 | 内容 |
|------|------|
| 被看见 | 看得越深，越容易被深层看见 |
| 边界削薄 | 自我与世界之间的皮肤变薄 |
| 梦境污染 | 梦不再是私人空间 |
| 系统标注异常 | 新手保护失效 |
| 庇护所依恋增强 | 庇护所开始不愿主角远行 |
| 候选人格靠近 | 神胎碎片投射诱惑 |

### 裂视等级

| 等级 | 能力 | 代价 |
|------|------|------|
| 0级·裂痕残光 | 偶尔看见系统标签闪烁 | 头痛、耳鸣 |
| 1级·错标识别 | 发现系统标注与现实不一致 | 梦中重复标签 |
| 2级·边界线感知 | 看见归属线 | 名字变轻 |
| 3级·规则注释 | 读取残缺骨刻语 | 梦境污染显著 |
| 4级·缝隙通行 | 发现隐藏门路 | 路会记住主角 |
| 5级·候选注视 | 识别其他候选人格纹路 | 神胎碎片开始回应 |
| 6级·临胎裂视 | 看见无光域整体代谢 | 不可完全恢复 |

---

## 庇护所活化机制

庇护所不是房屋，而是系统、文明与神胎争夺"家"概念的中间物。

### 三种结构叠合

1. **系统安全接口**——初始生存保护点
2. **文明边界钉**——门、床、箱、灯、灶将旧世界秩序钉入无光域
3. **晦能活化器官**——吸收晦晶、尸体、恐惧，逐渐长出类似生命体的反应

### 三种权

- **占有权**——系统层面的操作权
- **居住权**——门床箱墙的承认
- **内属权**——房屋将人视为自身一部分的危险状态

### 活化阶段（0级→7级）

从死屋到子宫方舟，每阶段都在深化人与房屋的边界重叠。

# 力量体系限制

---

## 裂视的限制

1. 不是透视眼、预知未来或读心术
2. 不是万能鉴定术或规则百科全书
3. 不是系统管理员权限
4. 不是完全可靠的真相提示
5. 不免疫污染
6. 裂视不告诉"正确答案是什么"，只提醒"这里有东西没有对齐"
7. 使用裂视会加速筛选进程

## 系统的限制

1. 系统不完整——只显示系统能编码的内容
2. 系统奖励不等于道德正确——系统奖励的是筛选价值
3. 系统交易只转移基础所有权，不保证边界完全切割
4. 排行榜不是荣誉，而是公共位置——会吞掉人
5. 系统可以错误——错栏、延迟、隐藏文字都是裂缝
6. 系统救人的目的不等于人的目的

## 庇护所的限制

1. 庇护所不能无限扩张——吞并废屋必须处理旧关系
2. 庇护所活化到一定程度会替主人做决定
3. 庇护所会形成"习惯"——重复行为会被固化
4. 庇护所可能产生"内属权"——危险状态
5. 庇护所不可能带走所有人——方舟伦理问题

## 力量成长限制

1. 必须有铺垫——不能突然获得能力
2. 必须有代价——每次使用消耗边界稳定性
3. 必须有失败风险——裂视可能误判
4. 必须有阶段性成长——不能跳过等级
5. 不得突然开挂解决所有问题
